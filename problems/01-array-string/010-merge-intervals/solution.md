# 解説

## 考え方

区間を開始位置で昇順に並べると、重なる可能性があるのは直前に確定した区間だけになります。現在の区間が直前の終了位置以内に始まるなら結合し、そうでなければ新しい区間として追加します。

## アルゴリズム

1. 空配列なら `[]` を返す
2. 各区間をコピーして、開始位置で昇順にソートする
3. `merged` に最初の区間を入れる
4. 2 番目以降の区間を順に見る
5. 現在区間の開始が最後の結合済み区間の終了以下なら、終了位置を大きい方に更新する
6. 重ならなければ、現在区間を `merged` に追加する

## 実装のポイント

- `intervals.map((interval) => [...interval])` により、入力の内側の配列を直接変更しない
- `current[0] <= last[1]` の場合は端点で接している区間も結合する
- 結合時は `last[1] = Math.max(last[1], current[1])` として終了位置を伸ばす

## 計算量

- 時間計算量: O(n log n)
- 空間計算量: O(n)
