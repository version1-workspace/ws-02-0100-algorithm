# 解説

## 考え方

`t` の文字を必要回数以上含む区間をスライディングウィンドウで探します。右端を広げて条件を満たしたら、左端を縮めながら最短の区間を更新します。

## アルゴリズム

1. `t.length > s.length` なら条件を満たせないので `""` を返す
2. `required` に `t` の各文字の必要回数を記録する
3. `right` を進めながら `window` の文字数を増やす
4. ある文字の窓内回数が必要回数と等しくなったら `formed` を増やす
5. `formed === required.size` の間、答えを更新しつつ `left` を進めて窓を縮める
6. 最短区間が見つかっていれば切り出して返し、なければ `""` を返す

## 実装のポイント

- `required.size` は、必要な文字の種類数を表す
- `formed` は、必要回数を満たした文字の種類数を表す
- 左端の文字を減らした結果、必要回数を下回ったら `formed` を減らして窓を無効にする
- `bestLength === Infinity` の場合は、条件を満たす区間が存在しない

## 計算量

- 時間計算量: O(|s| + |t|)
- 空間計算量: O(k)。`k` は `s` と `t` に現れる文字の種類数
