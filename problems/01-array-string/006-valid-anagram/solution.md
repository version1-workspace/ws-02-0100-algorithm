# 解説

## 考え方

アナグラムかどうかは、各文字の出現回数が一致するかで判定できます。`s` の文字数を数え、`t` の文字を見ながらその数を減らします。

## アルゴリズム

1. 長さが異なれば `false` を返す
2. `s` の各文字の出現回数を `counts` に記録する
3. `t` の各文字について、`counts` に存在しなければ `false` を返す
4. 回数が `1` ならその文字を削除し、それ以外なら回数を 1 減らす
5. 最後に `counts.size === 0` なら `true` を返す

## 実装のポイント

- `Map<string, number>` で文字ごとの残り回数を管理している
- 回数が `0` になった文字は `delete` して、最後に `counts.size` で判定できるようにしている
- 長さが違う場合は出現回数が一致しないため、先に除外する

## 計算量

- 時間計算量: O(n)
- 空間計算量: O(k)。`k` は文字の種類数
