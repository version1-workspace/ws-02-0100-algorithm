# 解説

## 考え方

3 つの数の合計を 0 にしたいので、1 つ目の数を固定し、残り 2 つをソート済み配列上の 2 ポインタで探します。ソートしておくと、合計が小さければ左を進め、大きければ右を戻す判断ができます。

## アルゴリズム

1. `nums` を昇順にコピーして `sorted` を作る
2. 先頭から `i` を動かし、同じ値を固定値にする重複はスキップする
3. `left = i + 1`、`right = sorted.length - 1` とする
4. 合計が `0` なら組を追加し、左右を進めて同じ値をスキップする
5. 合計が `0` より小さければ `left` を進め、大きければ `right` を戻す
6. すべて探索したら `result` を返す

## 実装のポイント

- `sorted = [...nums].sort(...)` として、入力配列自体は変更しない
- 固定値 `sorted[i]` の重複をスキップして、同じ三つ組を繰り返さない
- 解を追加した後も `left` と `right` の重複を飛ばして、結果の重複を防ぐ

## なぜ正しいか

固定した `sorted[i]` に対して、残り 2 数はソート済み範囲内で探します。合計が小さいときはより大きい値が必要なので `left` を進め、合計が大きいときはより小さい値が必要なので `right` を戻します。この操作で候補を漏れなく絞り込み、重複値だけをスキップするため、重複しない全ての三つ組を得られます。

## 計算量

- 時間計算量: O(n^2)
- 空間計算量: O(n)。ソート用コピーと結果配列を除いた追加領域は実装上 O(n)
