# 解説

## 考え方

各要素 `nums[i]` について、合計が `target` になる相手は `target - nums[i]` です。これまで見た値とそのインデックスを `Map` に保存しておけば、相手が存在するかを O(1) で確認できます。

## アルゴリズム

1. 値からインデックスを引くための `map` を用意する
2. 配列を左から順に見て、現在値に必要な `complement` を計算する
3. `map` に `complement` があれば、そのインデックスと現在の `i` を返す
4. なければ現在値と `i` を `map` に保存する
5. 問題条件では解が必ず存在するため通常はループ中に返る

## 実装のポイント

- `map` には「すでに見た値」だけを保存するため、同じ要素を 2 回使わない
- `map.get(complement)!` は、直前に `map.has(complement)` で存在確認しているので使える
- 末尾の `return []` は、問題条件に反する入力に対するフォールバック

## 計算量

- 時間計算量: O(n)
- 空間計算量: O(n)
