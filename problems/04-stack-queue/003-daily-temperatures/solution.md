# 解説

## 考え方

まだ答えが決まっていない日をスタックに入れておきます。新しい日の気温がスタック末尾の日より高ければ、その日が「次に暖かい日」になるため、待ち日数を確定できます。

スタックには気温そのものではなく日付のインデックスを入れます。答えは日付の差で計算するためです。

## アルゴリズム

1. 答え用の `result` をすべて `0` で初期化する
2. 未解決の日を入れる `stack` を用意する
3. `temperatures` を左から順に見る
4. 現在の気温が `stack` 末尾の日の気温より高い間、末尾の日を取り出して待ち日数を確定する
5. 現在の日を `stack` に積む
6. 最後まで残った日は、より高い気温の日がないので `0` のままにする

## 実装のポイント

- `stack` は気温が高くならずに残っている日のインデックスを保持します。
- `result[previousDay] = day - previousDay` で、何日待つかを計算しています。
- 同じ気温は「より高い気温」ではないため、比較は `>` です。

## 計算量

- 時間計算量: O(n)
- 空間計算量: O(n)
