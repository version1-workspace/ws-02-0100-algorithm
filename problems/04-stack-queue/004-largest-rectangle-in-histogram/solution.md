# 解説

## 考え方

ある棒を高さとする最大長方形は、その棒より低い棒に左右を挟まれるまで広げられます。単調増加スタックを使うと、「右側で初めて低くなる位置」が見つかったタイミングで、その棒を高さにした面積を確定できます。

## アルゴリズム

1. 棒のインデックスを入れる `stack` と `maxArea` を用意する
2. 左から順に `heights` を見る
3. 現在の高さがスタック末尾の高さより低い間、末尾の棒を取り出す
4. 取り出した棒の高さを `height` とする
5. 取り出した後のスタック末尾を左の境界、現在位置を右の境界として幅を計算する
6. `height * width` で面積を計算し、`maxArea` を更新する
7. 最後に高さ `0` の番兵を見たことにして、残った棒の面積も確定する

## 実装のポイント

- `stack` には高さが単調増加になるようにインデックスを積みます。
- `index === heights.length ? 0 : heights[index]` により、最後に番兵の高さ `0` を使っています。
- `leftBoundary` は、スタックが空なら `-1`、そうでなければ取り出した後のスタック末尾です。
- 幅は `index - leftBoundary - 1` です。現在位置は右側で初めて低い棒なので、長方形には含めません。

## なぜ正しいか

棒がスタックから取り出される瞬間、現在位置はその棒より右側で初めて低い位置です。また、取り出した後のスタック末尾は左側でその棒より低い位置です。したがって、その棒を高さにできる最大幅がその時点で確定し、各棒について最大の候補面積を漏れなく調べられます。

## 計算量

- 時間計算量: O(n)
- 空間計算量: O(n)
