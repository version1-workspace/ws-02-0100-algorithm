# 解説

## 考え方

括弧は最後に出てきた開き括弧から順に閉じる必要があります。これは LIFO の性質なので、開き括弧をスタックに積み、閉じ括弧が来たら直前の開き括弧と対応しているか確認します。

## アルゴリズム

1. 閉じ括弧から対応する開き括弧を引けるように `pairs` を用意する
2. 文字列を左から順に見る
3. 開き括弧なら `stack` に積む
4. 閉じ括弧なら `stack.pop()` した値が対応する開き括弧か確認する
5. 途中で対応しなければ `false` を返す
6. 最後に `stack` が空なら `true`、残っていれば `false` を返す

## 実装のポイント

- `pairs` は `")" -> "("` のように閉じ括弧をキーにしています。
- `stack.pop()` が `undefined` になる場合は、対応する開き括弧がないので比較に失敗して `false` になります。
- すべての閉じ括弧が対応していても、最後に開き括弧が残っていれば無効です。

## 計算量

- 時間計算量: O(n)
- 空間計算量: O(n)
