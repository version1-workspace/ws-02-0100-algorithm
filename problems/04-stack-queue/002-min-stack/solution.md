# 解説

## 考え方

通常のスタックだけでは、最小値を毎回探すと O(n) になります。そこで、各要素を積んだ時点での最小値も別のスタック `minimums` に積みます。これにより、現在の最小値は常に `minimums` の末尾から O(1) で取得できます。

## アルゴリズム

1. 値を保持する `values` と、各時点の最小値を保持する `minimums` を用意する
2. `push(value)` では `values` に値を積み、現在の最小値と `value` の小さい方を `minimums` に積む
3. `pop()` では `values` と `minimums` の末尾を同時に取り除く
4. `top()` では `values` の末尾を返す
5. `getMin()` では `minimums` の末尾を返す
6. 操作列を順番に処理し、各操作の返り値を配列にする

## 実装のポイント

- `minimums` は `values` と同じ長さを保つため、`pop` で最小値の履歴も同時に戻せます。
- 同じ最小値が複数回入る場合も、各時点の最小値を毎回積むので正しく扱えます。
- `MinStack` 操作が来たら新しい `MinStack` インスタンスを作り、返り値は `null` にします。
- 問題条件により、`pop`, `top`, `getMin` は空でないときだけ呼ばれます。

## 計算量

- 時間計算量: O(m)
- 空間計算量: O(m)

ただし `m` は操作数です。各操作自体は O(1) です。
