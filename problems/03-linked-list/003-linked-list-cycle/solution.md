# 解説

## 考え方

速いポインタ `fast` と遅いポインタ `slow` を同時に進めます。循環があれば、速いポインタはいずれ遅いポインタに追いつきます。循環がなければ、`fast` が末尾に到達します。

## アルゴリズム

1. `slow` と `fast` をどちらも `head` にする
2. `fast` と `fast.next` が存在する間だけループする
3. `slow` を 1 ノード、`fast` を 2 ノード進める
4. `slow === fast` になったら循環ありとして `true` を返す
5. ループを抜けたら循環なしとして `false` を返す

## 実装のポイント

- ノードの値ではなく、`slow === fast` という参照の一致で判定する
- `while (fast !== null && fast.next !== null)` により、循環がないリストで `null` 参照を避ける
- 空リストではループに入らず `false` を返す

## なぜ正しいか

循環がない場合、2 ノードずつ進む `fast` は必ず末尾に到達します。循環がある場合、`fast` は循環内で `slow` より毎回 1 ノード分だけ距離を詰めるため、有限回の移動で同じノードを指します。

## 計算量

- 時間計算量: O(n)
- 空間計算量: O(1)
