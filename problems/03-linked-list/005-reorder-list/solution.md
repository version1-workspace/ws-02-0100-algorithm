# 解説

## 考え方

`L0 -> L1 -> ... -> Ln` を `L0 -> Ln -> L1 -> Ln-1 ...` に並べ替えるため、リストを前半と後半に分け、後半を反転してから前半と交互にマージします。

## アルゴリズム

1. 空リストまたは 1 ノードならそのまま返す
2. `splitSecondHalf` で slow/fast pointer を使い、後半の先頭を切り出す
3. `reverseList` で後半リストを反転する
4. `mergeAlternately` で前半と反転後半を交互につなぐ
5. 元の `head` を返す

## 実装のポイント

- `splitSecondHalf` は `slow.next = null` として前半と後半を切り離す
- `reverseList` は各ノードの `next` を前のノードへ向け直す
- `mergeAlternately` では `next1` と `next2` を保存してからリンクを変更し、残りのリストを失わないようにしている
- ノードの値は変更せず、`next` の参照だけをつなぎ替えている

## 例で確認

`[1, 2, 3, 4, 5]` では、前半 `[1, 2, 3]` と後半 `[4, 5]` に分けます。後半を反転して `[5, 4]` にし、前半へ交互に差し込むと `[1, 5, 2, 4, 3]` になります。

## 計算量

- 時間計算量: O(n)
- 空間計算量: O(1)
