# 解説

## 考え方

`n` 段目に到達する直前は、`n - 1` 段目から1段進むか、`n - 2` 段目から2段進むかのどちらかです。
そのため、方法の数はフィボナッチ数列と同じ形で求められます。

## アルゴリズム

1. `n <= 2` の場合は、そのまま `n` を返す
2. `previous` を1段前、`current` を現在の段数までの方法数として持つ
3. 3段目から `n` 段目まで、`previous + current` で次の方法数を作る
4. 最後の `current` を返す

## 実装のポイント

- `previous` は `step - 2` 段目までの方法数、`current` は `step - 1` 段目までの方法数を表します。
- 配列を使わず、直前2つの値だけを更新するため、メモリを節約できます。

## 計算量

- 時間計算量: O(n)
- 空間計算量: O(1)
