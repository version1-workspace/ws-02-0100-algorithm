# 解説

## 考え方

各位置までの復号方法数は、最後の1桁を使う場合と最後の2桁を使う場合の合計です。
ただし `"0"` 単体は使えず、2桁として使えるのは `10` から `26` までです。

## アルゴリズム

1. 先頭が `"0"` なら復号できないので 0 を返す
2. `twoBack` と `oneBack` を 1 で初期化する
3. 2文字目以降を順に見る
4. 現在の文字が `"0"` でなければ、1桁として `oneBack` を足す
5. 直前との2桁が `10` から `26` なら、2桁として `twoBack` を足す
6. 状態を1つ進め、最後に `oneBack` を返す

## 実装のポイント

- `twoBack` は2つ前までの方法数、`oneBack` は1つ前までの方法数です。
- `"06"` は数値にすると 6 ですが、2桁の復号としては無効です。実装では `twoDigit >= 10` の条件で除外しています。
- 配列を使わず、直前2状態だけで更新しています。

## 計算量

- 時間計算量: O(n)
- 空間計算量: O(1)
