# 解説

## 考え方

2つの部分集合の合計を等しくするには、全体の合計の半分 `target` を作れる部分集合があれば十分です。
これは「各数を1回だけ使って `target` を作れるか」という 0/1 ナップサックとして考えます。

## アルゴリズム

1. 配列の合計 `total` を求める
2. `total` が奇数なら `false` を返す
3. `target = total / 2` とする
4. `dp[sum]` を「合計 `sum` を作れるか」として、`dp[0] = true` にする
5. 各 `num` について、`sum` を `target` から `num` まで逆順に更新する
6. `dp[target]` を返す

## 実装のポイント

- 逆順に更新することで、同じ `num` を同じ周回で複数回使うことを防ぎます。
- `dp[sum] || dp[sum - num]` により、今の数を使わない場合と使う場合を表します。

## 計算量

- 時間計算量: O(nums.length * target)
- 空間計算量: O(target)
