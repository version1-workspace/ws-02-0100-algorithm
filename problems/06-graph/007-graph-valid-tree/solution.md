# 解説

## 考え方

無向グラフが木である条件は、連結であり、循環を持たないことです。また、`n` 個のノードからなる木の辺数は必ず `n - 1` 本です。

実装ではまず辺数を確認し、その後 Union-Find で辺を 1 本ずつ追加します。すでに同じ連結成分に属する 2 ノードをつなごうとした場合、循環ができるため木ではありません。

## アルゴリズム

1. `edges.length !== n - 1` なら `false` を返す
2. `UnionFind` を `n` ノードで初期化する
3. 各辺 `[node1, node2]` について `union` する
4. `union` が `false` を返したら循環があるため `false` を返す
5. すべての辺を追加できたら `true` を返す

## なぜ正しいか

辺数が `n - 1` でないグラフは木になれません。辺数が `n - 1` の状態で循環がなければ、成分が複数に分かれる余地はありません。もし非連結なら、全体を木にするには少なくとも成分間をつなぐ追加の辺が必要になり、`n - 1` 本では足りないためです。

Union-Find は、辺を追加したときに同じ成分内の 2 ノードをつなぐかどうかを判定できます。同じ成分内なら既に経路があるので、その辺を足すと循環ができます。

## 実装のポイント

- `parents` は各ノードの親を持ち、根が連結成分の代表です。
- `ranks` は木の高さの目安で、低い木を高い木へつなぐことで探索を浅く保ちます。
- `find` は経路圧縮を行い、次回以降の代表探索を高速化します。
- `union` は結合できたら `true`、すでに同じ成分なら `false` を返します。

## 計算量

- 時間計算量: O(E α(N))
- 空間計算量: O(N)

ただし `N = n`、`E = edges.length`、`α` はアッカーマン関数の逆関数で、実用上はほぼ定数です。
