# 解説

## 考え方

隣接リストから一度 `GraphNode` のグラフを作り、そのグラフを DFS で複製します。元ノードと複製ノードの対応を `Map` で管理することで、同じノードを何度も作らず、辺の接続先も正しい複製ノードに向けられます。

最後に、複製したグラフを再び `number[][]` の隣接リストへ戻します。

## アルゴリズム

1. `buildGraph` で `adjList` から `GraphNode` の配列を作り、隣接関係を張る
2. `cloneNode` で開始ノードから DFS する
3. `clones` に元ノードから複製ノードへの対応を保存する
4. 隣接ノードの複製が未作成なら作成して `stack` に積む
5. 現在の複製ノードの `neighbors` に、隣接ノードの複製を追加する
6. `toAdjacencyList` で複製グラフを隣接リスト形式に変換して返す

## 実装のポイント

- `GraphNode` は `val` と `neighbors` を持つ内部表現です。
- `clones: Map<GraphNode, GraphNode>` により、元ノードをキーにして対応する複製ノードを取得できます。
- 空のグラフでは `buildGraph` と `cloneNode` が `null` を扱い、最終的に `[]` を返します。
- `toAdjacencyList` でも `visited` を使い、循環がある無向グラフでも同じノードを繰り返し処理しないようにしています。
- 実装は開始ノードから到達できる範囲を複製します。問題文では無向連結グラフとして与えられるため、この処理で全ノードをカバーできます。

## 計算量

- 時間計算量: O(V + E)
- 空間計算量: O(V + E)

ただし `V` はノード数、`E` は隣接リストに含まれる辺の数です。無向グラフでは各辺が両方向に現れるため、その分も入力サイズとして数えます。
