# 解説

## 考え方

前提科目の関係を有向グラフとして考えます。`[a, b]` は `b -> a` という辺で、`b` を終えると `a` を受講できるという意味です。

すべての科目を修了できるかは、この有向グラフに循環がないかで決まります。実装では入次数を使ったトポロジカルソートで、受講可能な科目から順に処理します。

## アルゴリズム

1. `graph[prerequisite]` に、その科目の後に受けられる科目を入れる
2. 各科目の未完了の前提数を `indegrees` に数える
3. 入次数が 0 の科目を `queue` に入れる
4. `queue` を先頭から順に処理し、完了数 `completed` を増やす
5. その科目を前提としている科目の入次数を 1 減らし、0 になったら `queue` に追加する
6. 最後に `completed === numCourses` なら `true`、そうでなければ `false` を返す

## なぜ正しいか

入次数が 0 の科目は、未完了の前提科目がないため必ず受講できます。受講済みの科目をグラフから取り除く操作は、その科目から出る辺の先の入次数を減らすことに対応します。

循環がある場合、循環内の科目は互いに前提を必要とするため、どこかの時点で入次数 0 になれず、全科目を処理できません。逆に全科目を処理できたなら、循環による行き詰まりはありません。

## 実装のポイント

- `graph` は前提科目から次に受けられる科目への隣接リストです。
- `indegrees[course]` は、まだ残っている前提科目の数です。
- `queue.shift()` は使わず、`index` を進める形で BFS 風に処理しているため、配列の先頭削除コストを避けています。

## 計算量

- 時間計算量: O(V + E)
- 空間計算量: O(V + E)

ただし `V = numCourses`、`E = prerequisites.length` です。
