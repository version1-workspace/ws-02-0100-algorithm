# 解説

## 考え方

最初は各ノードがそれぞれ別の連結成分です。辺で 2 つの異なる成分がつながるたびに、連結成分の数は 1 減ります。

実装では Union-Find を使い、各辺の両端を結合します。すでに同じ成分に属している辺は、連結成分数を変えません。

## アルゴリズム

1. `UnionFind` を `n` ノードで初期化する
2. 連結成分数 `componentCount` を `n` にする
3. 各辺 `[node1, node2]` について `union` する
4. 異なる成分を結合できた場合だけ `componentCount` を 1 減らす
5. 全ての辺を処理したら `componentCount` を返す

## 実装のポイント

- `parents` は各ノードの親を持ち、根が連結成分の代表です。
- `ranks` は木の高さの目安で、Union by Rank に使います。
- `find` は経路圧縮を行い、代表ノードの探索を高速化します。
- `union` が `false` の場合、両ノードはすでに同じ成分なので、連結成分数は減らしません。
- 辺を持たないノードは一度も結合されないため、単独の連結成分として残ります。

## 計算量

- 時間計算量: O(E α(N))
- 空間計算量: O(N)

ただし `N = n`、`E = edges.length`、`α` はアッカーマン関数の逆関数で、実用上はほぼ定数です。
