# 解説

## 考え方

木を一意に復元するには、値だけでなく `null` の位置も記録する必要があります。この実装では preorder DFS で「現在ノード、左部分木、右部分木」の順に文字列化し、空の子は `"null"` として保存します。

復元時も同じ preorder の順に読み進めれば、元の構造を再構築できます。

## アルゴリズム

1. `serialize` では `values` を用意する
2. DFS でノードが `null` なら `"null"` を追加する
3. ノードがあれば値を文字列で追加し、左、右の順に DFS する
4. `values` を `,` で結合して返す
5. `deserialize` では文字列を `,` で分割し、`index` で現在位置を管理する
6. 現在の値が `"null"` なら `null` を返す
7. 数値なら `TreeNode` を作り、左、右の順に再帰的に復元する

## 実装のポイント

- `null` も明示的に保存するため、左右どちらの子が空だったかを復元できます。
- `index` は `deserialize` 内の DFS 全体で共有され、読み終えた位置を進めていきます。
- 値は `String(node.val)` で保存し、復元時は `Number(value)` で数値に戻します。負数もそのまま扱えます。
- default export は `serialize` してから `deserialize` した木を返し、runner で検証できる形にしています。

## なぜ正しいか

preorder で各ノードの値と `null` をすべて記録すると、次に読む値が現在ノードか空ノードかを一意に判断できます。復元時に同じ順序で左部分木、右部分木を作るため、保存時と同じ構造と値の木が再構築されます。

## 計算量

- 時間計算量: O(n)
- 空間計算量: O(n)

文字列化した配列と復元時の分割配列に加え、再帰スタックとして O(h) を使いますが、全体では O(n) です。
