# 解説

## 考え方

二分探索木では、現在のノードより小さい値は左部分木、大きい値は右部分木にあります。`p` と `q` がどちらも現在ノードより小さければ共通祖先は左側、どちらも大きければ右側にあります。

値が左右に分かれた場合、または現在ノードが `p` か `q` のどちらかである場合、その現在ノードが最も低い共通祖先です。

## アルゴリズム

1. `current` を `root` にする
2. `current` が `null` でない間繰り返す
3. `p.val` と `q.val` がどちらも `current.val` より小さければ左へ進む
4. どちらも `current.val` より大きければ右へ進む
5. それ以外なら `current` を返す
6. 見つからなければ `null` を返す

## 実装のポイント

- 入力は有効な二分探索木で、`p` と `q` は必ず含まれるため、通常はループ内で答えが見つかります。
- `else` に入るのは、値が左右に分かれた場合、または `current` が `p` や `q` 自身の場合です。
- 再帰ではなく `while` で木を下るため、追加の再帰スタックを使いません。

## なぜ正しいか

`p` と `q` が同じ側にある間、現在ノードはまだ最も低い共通祖先ではなく、答えはその側の部分木にあります。初めて同じ側でなくなったノードは、両方を含む最も下の分岐点です。片方が現在ノード自身の場合も、ノード自身を子孫として扱うため、そのノードが答えになります。

## 計算量

- 時間計算量: O(h)
- 空間計算量: O(1)

ただし `h` は木の高さです。平衡なら O(log n)、偏っていれば O(n) です。
