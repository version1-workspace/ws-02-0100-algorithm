# 解説

## 考え方

二分探索木かどうかは、直下の子だけを見ても判定できません。あるノードは祖先から受け継いだ「この範囲に入っていなければならない」という制約も満たす必要があります。

そこで、各ノードに許される下限 `lowerBound` と上限 `upperBound` を渡しながら再帰します。

## アルゴリズム

1. `isValidBST` から `isValid(root, -Infinity, Infinity)` を呼ぶ
2. ノードが `null` なら有効として `true` を返す
3. `node.val` が下限以下、または上限以上なら `false` を返す
4. 左部分木は上限を `node.val` に更新して検証する
5. 右部分木は下限を `node.val` に更新して検証する
6. 左右どちらも有効なら `true` を返す

## 実装のポイント

- 比較は `<=` と `>=` なので、重複値は無効になります。
- 左部分木では `upperBound` が現在ノードの値になり、右部分木では `lowerBound` が現在ノードの値になります。
- 初期範囲に `-Infinity` と `Infinity` を使うことで、問題文の値域全体を自然に扱えます。

## なぜ正しいか

再帰で渡す範囲は、そのノードがすべての祖先との関係で満たすべき条件を表します。各ノードがこの範囲を満たし、左右の部分木にも更新後の範囲を渡して同じ条件を確認するため、木全体が二分探索木の条件を満たすかを判定できます。

## 計算量

- 時間計算量: O(n)
- 空間計算量: O(h)

ただし `n` はノード数、`h` は木の高さです。
