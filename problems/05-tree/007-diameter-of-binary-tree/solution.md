# 解説

## 考え方

あるノードを通る最長経路は、左部分木の高さと右部分木の高さを足したものです。これをすべてのノードで計算し、その最大値を直径として持っておきます。

高さの計算と直径の更新は同じ DFS の中で行えます。

## アルゴリズム

1. `diameter` を `0` で初期化する
2. `height(node)` で部分木の高さを返す
3. `node` が `null` なら高さ `0` を返す
4. 左右の高さを再帰的に求める
5. `leftHeight + rightHeight` で現在ノードを通る経路の辺数を計算し、`diameter` を更新する
6. 現在ノードを根とする高さとして `Math.max(leftHeight, rightHeight) + 1` を返す
7. DFS 後に `diameter` を返す

## 実装のポイント

- `height` はノード数ベースの高さを返します。
- 直径は辺の数なので、現在ノードを通る候補は `leftHeight + rightHeight` になります。
- 最長経路は根を通るとは限らないため、各ノードで `diameter` を更新します。

## なぜ正しいか

任意の最長経路には、その経路上で最も上にあるノードがあります。そのノードでは、経路は左部分木側と右部分木側へ伸びる形になるため、候補は左右の高さの和です。全ノードでこの候補を調べることで、根を通らない経路も含めて最大値を得られます。

## 計算量

- 時間計算量: O(n)
- 空間計算量: O(h)

ただし `n` はノード数、`h` は木の高さです。
