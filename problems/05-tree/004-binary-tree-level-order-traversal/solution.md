# 解説

## 考え方

階層ごとに上から順に値を集めるには BFS が合っています。キューに次に処理するノードを入れ、各階層の開始時点でのキューの長さを使うと、その階層だけをまとめて処理できます。

## アルゴリズム

1. `root` が `null` なら `[]` を返す
2. `result` と、根を入れた `queue` を用意する
3. キューが空になるまで繰り返す
4. 現在のキューの長さを `levelSize` として記録する
5. `levelSize` 個のノードを取り出し、値を `level` に入れる
6. 取り出したノードの左子、右子があればキューに追加する
7. 1階層分の `level` を `result` に追加する

## 実装のポイント

- `levelSize` を先に固定することで、あとから追加した子ノードは次の階層として処理されます。
- `queue.shift()!` で先頭から取り出し、左から右の順序を保っています。
- 空の木ではキューを作らず、すぐ空配列を返します。

## 計算量

- 時間計算量: O(n)
- 空間計算量: O(n)

`shift()` は配列の先頭削除なので JavaScript の実装上は追加コストが出る場合がありますが、この `solution.ts` はキューとして配列を使う BFS の方針です。
