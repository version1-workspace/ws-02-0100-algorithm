# 解説

## 考え方

上位 `k` 個の大きい値だけを保持できれば、その中で最小の値が「k番目に大きい値」です。
そこでサイズ `k` の Min Heap を使い、不要になった小さい値を捨てます。

## アルゴリズム

1. 空の `MinHeap` を用意する
2. `nums` の各値を heap に追加する
3. heap のサイズが `k` を超えたら最小値を取り除く
4. 最後に heap の先頭 `peek()` を返す

## なぜ正しいか

heap のサイズを常に最大 `k` に保ち、`k` 個を超えた時点で最小値を捨てています。
そのため処理後の heap には配列全体の上位 `k` 個だけが残り、その中の最小値が重複を含めた `k` 番目に大きい要素です。

## 実装のポイント

- `MinHeap` は配列 `values` で二分ヒープを表現しています。
- `bubbleUp` と `bubbleDown` で、親が子以下になる Min Heap の性質を保ちます。
- `peek()!` は、制約上 `k >= 1` で heap が空にならないため安全です。

## 計算量

- 時間計算量: O(n log k)
- 空間計算量: O(k)
