# 解説

## 考え方

各リストは昇順なので、各リストの先頭候補の中で最小のノードを選び続ければ、全体も昇順にマージできます。
最小候補を効率よく取り出すために Min Heap を使います。

## アルゴリズム

1. ノードの `val` で比較する `MinHeap<ListNode>` を用意する
2. 各リストの先頭ノードを heap に入れる
3. dummy ノードと `tail` を用意する
4. heap から最小ノードを取り出し、結果リストの末尾につなぐ
5. 取り出したノードに次のノードがあれば heap に入れる
6. 最後に `tail.next = null` とし、`dummy.next` を返す

## 実装のポイント

- `dummy` を使うことで、結果リストの先頭を特別扱いせずに済みます。
- `next` を先に保存してからノードをつなぐため、元リストの次候補を失いません。
- 最後に `tail.next = null` として、不要な接続が残らないようにしています。

## 計算量

- 時間計算量: O(N log k) ただし `N` は全ノード数、`k` はリスト数
- 空間計算量: O(k)
