# 解説

## 考え方

最も多く出現するタスクが、必要な待機時間を決める中心になります。
最大出現回数のタスクを並べ、その間に長さ `n` の隙間を作ると、最低限必要な枠の数を計算できます。

## アルゴリズム

1. `Map` で各タスクの出現回数を数える
2. 最大出現回数 `maxFrequency` を求める
3. 最大出現回数を持つタスクの種類数 `maxFrequencyCount` を数える
4. `(maxFrequency - 1) * (n + 1) + maxFrequencyCount` で必要枠を計算する
5. 実タスク数 `tasks.length` と必要枠の大きい方を返す

## なぜ正しいか

最大頻度のタスク同士の間には、少なくとも `n` 個分の間隔が必要です。
`maxFrequency - 1` 個の区間を作り、それぞれを長さ `n + 1` の枠として考えると、最後に最大頻度タスクの種類数を足した値が idle を含む最小枠になります。
ただし他のタスクで隙間をすべて埋められる場合は idle が不要なので、答えは `tasks.length` 以上になります。

## 実装のポイント

- `maxFrequencyCount` は、最大頻度のタスクが複数ある場合に最後の枠へ並べる数です。
- Heap を使わず、頻度から直接答えを計算する greedy 解法です。
- `n = 0` の場合も `Math.max(tasks.length, frameLength)` で実タスク数が返ります。

## 計算量

- 時間計算量: O(tasks.length)
- 空間計算量: O(u) ただし `u` はタスク種類数
