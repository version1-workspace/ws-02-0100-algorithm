# 解説

## 考え方

中央値をすぐ取り出せるように、小さい半分を Max Heap、大きい半分を Min Heap で管理します。
小さい側の最大値と大きい側の最小値が中央値の候補になります。

## アルゴリズム

1. `MedianFinder` を作り、操作列を順に処理する
2. `addNum` では、値を `lower` または `upper` に追加する
3. `rebalance` で `lower` のサイズが `upper` と同じか1つ多い状態に保つ
4. `findMedian` では、奇数個なら `lower.peek()`、偶数個なら両 heap の先頭の平均を返す
5. 各 `findMedian` の結果だけを配列に追加して返す

## なぜ正しいか

`lower` には小さい半分、`upper` には大きい半分を入れ、サイズ差を高々1に保ちます。
この状態では、要素数が奇数なら小さい半分の最大値が中央、偶数なら小さい半分の最大値と大きい半分の最小値の平均が中央値です。

## 実装のポイント

- `lower` は比較関数を逆にした Max Heap、`upper` は通常の Min Heap として使っています。
- `rebalance` により、`upper` の方が大きくならないようにしています。
- 問題条件で `findMedian` は少なくとも1つ追加された後に呼ばれるため、`peek()!` が使われています。

## 計算量

- 時間計算量: `addNum` は O(log n)、`findMedian` は O(1)
- 空間計算量: O(n)
