# 解説

## 考え方

回転された配列でも、中央で分けると左右どちらか一方は必ず昇順に並んでいます。
昇順になっている側に `target` が含まれるかを判定し、含まれない側を捨てます。

## アルゴリズム

1. `left` と `right` で探索範囲を持つ
2. `mid` を求め、`nums[mid] === target` なら返す
3. `nums[left] <= nums[mid]` なら左半分が昇順
4. 左半分に `target` が入るなら `right = mid - 1`、そうでなければ `left = mid + 1`
5. 右半分が昇順なら、同様に右半分に入るかで範囲を更新する
6. 見つからなければ `-1` を返す

## なぜ正しいか

重複がない回転済み昇順配列では、任意の `mid` に対して `[left, mid]` か `[mid, right]` の少なくとも片方は通常の昇順区間です。
昇順区間では端の値から `target` が含まれるかを判定できるため、答えを含まない半分だけを確実に捨てられます。

## 実装のポイント

- `nums[left] <= nums[mid]` で左側がソート済みかを判定します。
- 範囲判定では、`mid` はすでに一致確認済みなので `< nums[mid]` や `nums[mid] < target` としています。

## 計算量

- 時間計算量: O(log n)
- 空間計算量: O(1)
