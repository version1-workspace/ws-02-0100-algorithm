# 解説

## 考え方

最小値は、回転によって値が小さくなる境目にあります。
`nums[mid]` と右端 `nums[right]` を比べると、最小値が右側にあるか、左側を含む範囲にあるかを判断できます。

## アルゴリズム

1. `left = 0`、`right = nums.length - 1` とする
2. `left < right` の間、`mid` を求める
3. `nums[mid] > nums[right]` なら最小値は `mid` より右なので `left = mid + 1`
4. そうでなければ最小値は `mid` を含む左側なので `right = mid`
5. `left` と `right` が一致した位置の値を返す

## なぜ正しいか

`nums[mid] > nums[right]` なら、`mid` から右端までの区間内に回転の境目があります。
逆に `nums[mid] <= nums[right]` なら、`mid` から右端までは昇順なので、最小値は `mid` 自身かその左側にあります。

## 実装のポイント

- `right = mid` としているのは、`nums[mid]` 自体が最小値の可能性を残すためです。
- 重複がない前提なので、比較だけで探索範囲を半分にできます。

## 計算量

- 時間計算量: O(log n)
- 空間計算量: O(1)
