# 解説

## 考え方

2つの配列を実際にマージせず、左半分と右半分に分ける位置を二分探索します。
左半分のすべての値が右半分のすべての値以下になれば、中央値は境界の値から求められます。

## アルゴリズム

1. 短い方の配列を `nums1` にする
2. 全体の左半分の要素数 `halfLength` を求める
3. `nums1` の分割位置 `partition1` を二分探索する
4. `partition2 = halfLength - partition1` で `nums2` 側の分割位置を決める
5. 左側最大値と右側最小値をそれぞれ求める
6. `maxLeft1 <= minRight2` かつ `maxLeft2 <= minRight1` なら中央値を返す
7. 条件を満たさない場合は、`partition1` を左右に動かす

## なぜ正しいか

左半分の要素数を固定し、さらに左側の最大値が右側の最小値以下になれば、全体をソートしたときの左半分と右半分が正しく分かれています。
その境界の最大値と最小値だけを見れば、奇数長・偶数長どちらの中央値も求められます。

## 実装のポイント

- `Number.NEGATIVE_INFINITY` と `Number.POSITIVE_INFINITY` を使い、分割位置が端にある場合も同じ比較式で扱います。
- 短い配列側を二分探索することで、`partition2` が扱いやすくなります。
- 末尾の `return 0` は、問題条件どおりソート済み配列が与えられる限り到達しない想定のフォールバックです。

## 計算量

- 時間計算量: O(log min(m, n))
- 空間計算量: O(1)
