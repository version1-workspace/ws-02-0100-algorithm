# 解説

## 考え方

食べる速度 `k` が大きいほど必要時間は短くなります。
この単調性を使い、「`h` 時間以内に食べ切れるか」を判定しながら最小の `k` を二分探索します。

## アルゴリズム

1. 探索範囲を `left = 1`、`right = max(piles)` とする
2. 中央の速度 `mid` を試す
3. 各山に必要な時間 `ceil(pile / mid)` を合計する
4. 合計時間が `h` 以下なら、より遅い速度も試すため `right = mid`
5. 間に合わないなら `left = mid + 1`
6. 最後に `left` を返す

## なぜ正しいか

ある速度で時間内に食べ切れるなら、それより速い速度でも必ず食べ切れます。
つまり「可能かどうか」は速度に対して false から true に切り替わる単調な条件なので、最初に true になる速度を二分探索で求められます。

## 実装のポイント

- `Math.ceil(pile / mid)` で、1つの山を食べ切るのに必要な時間を計算します。
- `right = mid` として、現在の速度が答えである可能性を残します。

## 計算量

- 時間計算量: O(piles.length * log max(piles))
- 空間計算量: O(1)
